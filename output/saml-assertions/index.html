SAML assertions

SAML assertions are XML-based statements that convey information about a subject, typically a user, in the context of authentication and authorization.

Description

SAML (Security Assertion Markup Language) assertions are a fundamental component of SAML, which is an open standard for exchanging authentication and authorization data between parties, particularly between an identity provider (IdP) and a service provider (SP). SAML assertions carry information about a user, such as their identity, attributes, and authorization decisions. These assertions are signed to ensure their integrity and authenticity. SAML assertions can be classified into three types: authentication assertions, which confirm that the user has been authenticated; attribute assertions, which provide additional information about the user (like roles or permissions); and authorization decision assertions, which assert whether a user is authorized to access a resource. In an Identity and Access Management (IAM) context, SAML assertions are used to facilitate Single Sign-On (SSO) capabilities, allowing users to authenticate once and gain access to multiple services without needing to log in repeatedly. This enhances user experience while maintaining security and control over access to resources.

Examples

  • A user logs into a corporate application using SSO; a SAML assertion from the IdP is sent to the SP to grant access.
  • A service provider uses a SAML assertion to verify user attributes to customize user experience based on role.

Additional Information

  • SAML assertions are typically transmitted over secure channels using HTTPS.
  • SAML is widely used in enterprise environments for federated identity management and cloud service access.
